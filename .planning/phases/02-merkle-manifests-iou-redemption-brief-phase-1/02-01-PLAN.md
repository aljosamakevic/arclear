<!-- The construction spec BOTH implementations must follow (RESEARCH Pattern 2/3). -->

Construction (locked D-01..D-05 + RFC 6962 prefixes per research Q3):
```
leaf:   L_i = keccak256(0x00 ‖ id_i)         ids strictly ascending, unique, lowercase hex
node:   N   = keccak256(0x01 ‖ left ‖ right) ordered concatenation, NEVER sorted
level:  pair (2j, 2j+1) → parent j; odd level width → last node promotes UNCHANGED
root:   single remaining node; EMPTY list → sentinel keccak256("0x")
```

Proof encoding (planner-locked from research Pattern 3 — the Solidity structs in 02-02 mirror these exactly):
```typescript
export interface InclusionProof {
  leaf: Hex;          // raw IOU id (pre-leaf-hash)
  index: number;      // 0-based position in sorted leaf list
  leafCount: number;  // total leaves in that round's manifest
  siblings: Hex[];    // bottom-up; promotion levels consume no sibling
}
export type NonInclusionKind = "belowFirst" | "aboveLast" | "bracket"; // Solidity enum order 0,1,2
export interface NonInclusionProof {
  kind: NonInclusionKind;
  a: InclusionProof;  // belowFirst: first leaf | aboveLast: last leaf | bracket: lower neighbor
  b: InclusionProof;  // bracket only: upper neighbor (ignored otherwise; use a copy of `a` as placeholder)
}
```

Inclusion verification walk (identical both sides): start `i = index`, `w = leafCount`, `h = leafHash(leaf)`; while `w > 1`: if `i` odd → consume sibling on the LEFT; else if `i != w - 1` → consume sibling on the RIGHT; else promote (no sibling); then `i >>= 1; w = (w + 1) >> 1`. Valid iff `index < leafCount`, ALL siblings consumed exactly, and result equals root.

Non-inclusion rules: empty root sentinel ⇒ non-inclusion trivially holds; belowFirst ⇒ verify `a` at index 0 and `id < a.leaf`; aboveLast ⇒ verify `a` at index `leafCount - 1` and `id > a.leaf`; bracket ⇒ verify both, `a.leafCount == b.leafCount`, `b.index == a.index + 1`, `a.leaf < id < b.leaf`. Strict inequalities everywhere.
