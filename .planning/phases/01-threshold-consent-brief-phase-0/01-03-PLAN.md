```typescript
export type ConsentOutcome =
  | { kind: "consent"; signature: Hex }
  | { kind: "refusal"; reason: string };
// timeout is NOT a provider outcome; it is the coordinator's deadline firing

export type ConsentProvider = (proposal: RoundProposal, excluded: Address[]) => Promise<ConsentOutcome>;

export async function collectConsents(
  proposal: RoundProposal, excluded: Address[],
  providers: Map<string, ConsentProvider>,  // lowercase addr -> provider
  windowMs: number,
): Promise<{ consents: Map<string, Hex>; refused: { address: Address; reason: string }[]; timedOut: Address[] }>;

// Two-pass driver, chain-free: tests inject providers + a recording submit callback
export async function attemptRound(args: {
  hub: Address; roundNonce: bigint; openIous: SignedIou[];
  settledIds: ReadonlySet<Hex>; providers: Map<string, ConsentProvider>;
  windowMs: number; now: bigint; chainId?: number;
  submit: (proposal: RoundProposal, signatures: Hex[]) => Promise<Hex>;
}): Promise<RoundAttemptOutcome>;
// RoundAttemptOutcome discriminates: settled (proposal, signatures, txHash, excluded, passCount)
//   | aborted (reason, excluded, passCount) | empty (nothing to net / <2 participants on pass 1)
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| member → coordinator | Consent/refusal/stall responses are untrusted, unbounded-latency input |
| coordinator → members (pass 2) | Rebuilt proposal + excluded list — coordinator may equivocate between passes |
| coordinator → contract | Submitted round must be individually unanimous |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-09 | Denial of Service | stall to block settlement | mitigate | Two-pass exclude-and-recompute; hard 2-pass cap (D-03); unconditional re-inclusion (D-04) — cost bounded to latency, proven by Task 3 tests |
| T-01-10 | Spoofing | pass-1 signature replay on pass-2 digest | mitigate | attemptRound never carries signatures across passes (Task 2 acceptance); digest differs; contract BadSignature backstop |
| T-01-11 | Tampering | coordinator equivocation between passes (both fully signed) | mitigate | Both sets are individually unanimous → either is safe; shared roundNonce means at most one executes (WrongRoundNonce → clean abort, Pitfall 4); documented in Plan 05 |
| T-01-12 | Tampering | double-settle of excluded IOUs | mitigate | settledIds mutated only on confirmed settlement; Task 3 disjoint-manifest property (CONS-04) |
| T-01-13 | Repudiation | fake-timeout censorship by coordinator | accept | Victim's funds cannot move without their signature; anyone can run a coordinator; cost is latency only — documented in griefing analysis (Plan 05) |
| T-01-14 | Elevation | miss-counter inflation by malicious coordinator | accept | Counter is coordinator-local with zero protocol effect until Phase 2; flagged as Phase 2 design input |
| T-01-SC | Tampering | npm/pip/cargo installs | accept | No package installs in this plan |
</threat_model>
